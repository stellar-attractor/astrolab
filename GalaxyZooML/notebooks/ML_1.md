Это CNN для классификации морфологии галактик на 3 класса:
	•	Elliptical
	•	Spiral
	•	Artifact

Если по слоям и по смыслу, то модель такая.

⸻

Общая идея

На вход подается картинка галактики:

128 × 128 × 3

Дальше модель:
	1.	слегка аугментирует изображение,
	2.	извлекает визуальные признаки через сверточные блоки,
	3.	сжимает карту признаков в компактный вектор,
	4.	классифицирует объект в один из 3 классов.

⸻

Архитектура по слоям

1. Input

tf.keras.layers.Input(shape=(128, 128, 3))

Вход:
	•	RGB изображение
	•	размер 128×128

⸻

2. Data Augmentation

data_augmentation = tf.keras.Sequential([
    RandomFlip("horizontal"),
    RandomRotation(0.15),
    RandomZoom(0.10),
    RandomContrast(0.15),
])

Что делает:
	•	случайно отражает изображение по горизонтали,
	•	немного поворачивает,
	•	слегка масштабирует,
	•	немного меняет контраст.

Зачем:
	•	чтобы модель не запоминала конкретные картинки,
	•	чтобы лучше обобщала на новые изображения.

Важно: это не “обучаемые” слои в классическом смысле, а слой преобразования входа во время train.

⸻

3. Первый сверточный блок

Conv2D(32, 3, padding="same", use_bias=False)
BatchNormalization()
Activation("relu")
MaxPooling2D()

Conv2D(32, 3, padding="same", use_bias=False)
	•	32 фильтра
	•	ядро 3×3
	•	same сохраняет пространственный размер
	•	use_bias=False, потому что дальше сразу идет BatchNorm

Назначение:
	•	извлекает базовые локальные признаки:
	•	края,
	•	пятна,
	•	яркостные переходы,
	•	простые текстуры.

BatchNormalization()
	•	стабилизирует распределение активаций,
	•	ускоряет и стабилизирует обучение.

Activation("relu")
	•	добавляет нелинейность,
	•	позволяет модели учить сложные зависимости.

MaxPooling2D()
	•	уменьшает пространственное разрешение в 2 раза.

Примерно:

128×128×32 → 64×64×32


⸻

4. Второй сверточный блок

Conv2D(64, 3, padding="same", use_bias=False)
BatchNormalization()
Activation("relu")
MaxPooling2D()

Теперь:
	•	фильтров уже 64,
	•	модель учит более сложные паттерны.

Что начинает видеть:
	•	более крупные структуры,
	•	дуги,
	•	дисковые формы,
	•	распределение света,
	•	зачатки спиральных рукавов.

После pooling:

64×64×64 → 32×32×64


⸻

5. Третий сверточный блок

Conv2D(128, 3, padding="same", use_bias=False)
BatchNormalization()
Activation("relu")

Здесь:
	•	128 фильтров,
	•	без pooling после блока.

Назначение:
	•	построение более абстрактных и высокоуровневых признаков:
	•	компактность профиля,
	•	наличие/отсутствие выраженной структуры,
	•	спиральность,
	•	асимметрия,
	•	признаки артефактов.

⸻

6. GlobalAveragePooling2D

GlobalAveragePooling2D()

Это важный слой.

Он заменяет Flatten().

Что делает:
	•	берет каждую feature map и усредняет её по всей площади,
	•	превращает тензор вида примерно:

32×32×128

в вектор длины:

128

Почему это хорошо:
	•	меньше параметров, чем у Flatten,
	•	меньше риск переобучения,
	•	модель больше смотрит на наличие признака, чем на его точную координату,
	•	для задач морфологии это часто разумно.

⸻

7. Dropout

Dropout(0.35)

Во время обучения случайно выключает 35% признаков.

Зачем:
	•	уменьшает переобучение,
	•	заставляет сеть не полагаться на один узкий набор нейронов.

⸻

8. Полносвязный скрытый слой

Dense(64, activation="relu")

Это уже классификационная “голова”.

Что делает:
	•	берет 128 агрегированных признаков,
	•	комбинирует их в более компактное представление,
	•	подготавливает решение о классе.

Можно интерпретировать как слой, который объединяет:
	•	“насколько объект гладкий”,
	•	“есть ли выраженная структура/диск”,
	•	“похоже ли это вообще на галактику”.

⸻

9. Второй Dropout

Dropout(0.25)

Дополнительная регуляризация перед финальным классификатором.

⸻

10. Выходной слой

Dense(3, activation="softmax")

Финальный классификатор на 3 класса.

На выходе получаем вероятности:

[ P(Elliptical), P(Spiral), P(Artifact) ]

Их сумма = 1.

⸻

Что модель фактически учит

По смыслу сеть пытается выделить такие визуальные признаки:

Для Elliptical
	•	гладкое распределение света,
	•	отсутствие выраженных деталей,
	•	более симметричный и однородный профиль.

Для Spiral
	•	наличие структуры,
	•	диск,
	•	рукава,
	•	неоднородность яркости,
	•	более сложная морфология.

Для Artifact
	•	всё, что похоже не на нормальную галактику:
	•	звезды,
	•	шум,
	•	дефекты,
	•	артефакты наблюдений/разметки.

⸻

Почему эта архитектура разумна

Это уже не совсем “самый базовый baseline”, а умеренно усиленный компактный CNN:

по сравнению с простейшей версией у тебя добавлены:
	•	augmentation,
	•	batch normalization,
	•	третий conv block,
	•	global average pooling вместо flatten,
	•	dropout,
	•	callbacks (EarlyStopping, ReduceLROnPlateau).

То есть это хорошая компактная архитектура для старта:
	•	не слишком тяжёлая,
	•	достаточно устойчивая,
	•	подходит для ноутбука и для первой осмысленной классификации.

⸻

Архитектура в одной строке

Input
→ Augmentation
→ Conv(32) + BN + ReLU + MaxPool
→ Conv(64) + BN + ReLU + MaxPool
→ Conv(128) + BN + ReLU
→ GlobalAveragePooling
→ Dropout
→ Dense(64, ReLU)
→ Dropout
→ Dense(3, Softmax)


⸻

Что это за класс модели

Формально это:
	•	supervised image classifier
	•	CNN
	•	multi-class classification
	•	lightweight custom Keras model

Не transfer learning, не ResNet, не EfficientNet — а собственная компактная сверточная сеть.

⸻

Ограничения модели

Она простая, поэтому:
	•	может не очень хорошо ловить тонкие морфологические различия,
	•	чувствительна к шумным labels,
	•	не использует предобученные визуальные признаки,
	•	не учитывает confidence голосов Galaxy Zoo, только argmax.

Но для baseline/education/outreach — очень хороша.
